BOM : Browser Object Model

  • BOM 이란?

    브라우저의 창이나 프레임을 프로그래밍적으로 제어할 수 있게 해주는 객체 모델입니다.
    웹 페이지의 내용(content)과는 독립적으로 브라우저의 기능(새 창 띄우기, 주소 이동, 화면 크기 조절 등)을 제어합니다.

  • BOM의 주요 객체

    window: BOM의 최상위 객체로, 브라우저의 창을 나타냅니다. 전역 변수와 함수를 포함합니다. (e.g., alert(), setTimeout())

    navigator: 브라우저의 이름, 버전, 언어, 위치 정보 등을 제공합니다.

    screen: 사용자의 디스플레이 화면 정보(해상도, 색상 깊이 등)를 제공합니다.

    location: 현재 페이지의 URL 정보를 담고 있으며, 다른 페이지로 이동시킬 수 있습니다. (e.g., location.href = '...')

    history: 브라우저의 방문 기록을 제어합니다. (e.g., history.back(), history.forward())

  • 사용 시 알아야 할 점

    BOM은 W3C 같은 표준화 기구에서 정의한 공식 표준이 아닙니다. 그래서 브라우저 제조사마다 일부 동작이 다를 수 있었지만, 최근에는 대부분의 기능이 통일되었습니다.

DOM : Document Object Model

  • DOM 이란?

    HTML, XML 문서를 위한 프로그래밍 인터페이스입니다.
    문서를 트리 구조의 노드(Node)와 객체(Object)로 표현하여, JavaScript와 같은 스크립트 언어가 문서의 구조, 스타일, 내용 등을 동적으로 변경할 수 있게 해줍니다.

  • DOM의 주요 특징

    document: DOM 트리의 최상위 진입점(root) 객체입니다. HTML 문서 전체를 나타냅니다.

    문서의 모든 요소(element), 속성(attribute), 텍스트를 각각의 객체로 다룰 수 있습니다.

    DOM API를 통해 HTML 요소를 선택(querySelector), 생성(createElement), 수정(innerHTML), 삭제(remove)할 수 있습니다.

  • 사용 시 알아야 할 점

    DOM은 W3C 표준이므로 모든 브라우저에서 동일하게 동작합니다.

    DOM을 직접 조작하여 화면을 변경하는 것은 렌더링 과정(리플로우, 리페인트)을 다시 유발하므로, 잦은 변경은 성능 저하의 원인이 될 수 있습니다.
